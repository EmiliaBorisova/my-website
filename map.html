UW Campus Map

Overview

During the Spring quarter of my sophomore year, I developed a React application that could find the shortest path between two buildings on UW’s campus and draw the directions on the map. The user can enter the names of the buildings using dropdown menus, or they can alternatively click on the buildings for their query.

The first step was to develop a generic Graph data structure using Java so that the class could be used at multiple points throughout the project with different data types. For example, the graph was used in Dijkstra’s algorithm with numbers, and it was also used in a Breadth First Search (BFS) algorithm in a testing application that worked with strings. The second step was to use the Model, View, Controller design pattern for the development of the application. The Graph was populated with information about the campus buildings from an excel file and then the React framework was used to implement the UI and listen to user input. Once a user entered a query of two buildings, Dijkstra’s was used to find the shortest path between two building nodes and the Spark web application framework communicated the results to the front-end. Throughout the process, multiple testing strategies were utilized, namely script tests as well as black box and white box JUnit testing.

Technologies Used

  • Java
    • For the generic Graph data structure, Dijkstra’s algorithm, and BFS algorithm
  • Spark
    • Web framework that collected the results from Dijkstra’s algorithm and sent it to the front-end
  • React
    • Framework used for the UI and sending requests to the back-end
